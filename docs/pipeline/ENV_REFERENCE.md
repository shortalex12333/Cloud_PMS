Environment Reference (CI vs Render)
====================================

CI (GitHub Actions)
- BASE_URL → API_BASE
- MASTER_SUPABASE_URL
- MASTER_SUPABASE_ANON_KEY
- MASTER_SUPABASE_SERVICE_ROLE_KEY (admin API for user create)
- TENANT_SUPABASE_URL
- TENANT_SUPABASE_SERVICE_ROLE_KEY (doc_metadata insert/read‑only tests)
- TEST_USER_YACHT_ID (YACHT_ID)
- STAGING_USER_PASSWORD
- Optional: STAGING_CREW_EMAIL, STAGING_HOD_EMAIL, STAGING_CAPTAIN_EMAIL (or CREATE_USERS=true)

Render (Staging Service)
- yTEST_YACHT_001_SUPABASE_URL
- yTEST_YACHT_001_SUPABASE_SERVICE_KEY
- DEFAULT_YACHT_CODE=yTEST_YACHT_001
- MASTER_SUPABASE_URL
- MASTER_SUPABASE_SERVICE_KEY
- MASTER_SUPABASE_JWT_SECRET
- FEATURE_CERTIFICATES=true
- UI_CERTIFICATES=true
- Note: TENNANT_* typo is unused; TENANT_* names are CI-only, not Render.
