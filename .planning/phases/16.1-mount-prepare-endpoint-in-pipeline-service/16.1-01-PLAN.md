**Phase:** 16.1 — Mount /prepare endpoint in pipeline_service
**Plan:** 01
**Created:** 2026-03-02
**Status:** COMPLETE

---

## Objective

Fix GAP-001: Mount the `/v1/actions/prepare` endpoint so it's accessible at `localhost:8000/v1/actions/prepare` instead of returning 404.

---

## Problem Statement

The `/v1/actions/prepare` endpoint exists in `action_router/router.py:1248` but returns 404 because:
- Only `p0_actions_router` from `routes/p0_actions_routes.py` is mounted in `pipeline_service.py:199`
- The `action_router.router` containing `/prepare` is **never mounted**

---

## Requirements Addressed

| Requirement | Description |
|-------------|-------------|
| GAP-001 | /prepare endpoint must be accessible (CRITICAL - blocks all prefill) |
| PREFILL-01 | Prefill endpoint must accept NLP query and return prefilled form |

---

## Implementation Approach

**Option chosen:** Move `/prepare` endpoint to `p0_actions_routes.py` by importing models and recreating the route.

**Why not mount `action_router.router`?**
- Both routers have `/execute` endpoint — would cause route conflicts
- Cleaner to have single source of truth for `/v1/actions/*`
