# Plan 00-02: Extend Tailwind Config with Semantic Tokens

## Objective

Extend tailwind.config.ts to map semantic color classes to CSS custom properties, enabling usage like `bg-surface-base`, `text-txt-primary`, `text-brand-interactive`.

## Tasks

<task id="1">
Read current apps/web/tailwind.config.ts to understand existing configuration
</task>

<task id="2">
Extend the `theme.extend.colors` section with semantic token mappings:
```js
colors: {
  brand: {
    ambient: 'var(--brand-ambient)',
    interactive: 'var(--brand-interactive)',
    hover: 'var(--brand-hover)',
    muted: 'var(--brand-muted)',
  },
  status: {
    critical: 'var(--status-critical)',
    'critical-bg': 'var(--status-critical-bg)',
    warning: 'var(--status-warning)',
    'warning-bg': 'var(--status-warning-bg)',
    success: 'var(--status-success)',
    'success-bg': 'var(--status-success-bg)',
    neutral: 'var(--status-neutral)',
    'neutral-bg': 'var(--status-neutral-bg)',
  },
  surface: {
    base: 'var(--surface-base)',
    primary: 'var(--surface-primary)',
    elevated: 'var(--surface-elevated)',
    hover: 'var(--surface-hover)',
    active: 'var(--surface-active)',
    border: 'var(--surface-border)',
    'border-subtle': 'var(--surface-border-subtle)',
  },
  txt: {
    primary: 'var(--text-primary)',
    secondary: 'var(--text-secondary)',
    tertiary: 'var(--text-tertiary)',
    disabled: 'var(--text-disabled)',
    inverse: 'var(--text-inverse)',
  },
}
```
</task>

<task id="3">
Add spacing extensions using CSS variables:
```js
spacing: {
  '1': 'var(--space-1)',
  '2': 'var(--space-2)',
  // ... etc
}
```
</task>
