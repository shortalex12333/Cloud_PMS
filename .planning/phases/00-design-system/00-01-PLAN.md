# Plan 00-01: Implement Design Tokens CSS

## Objective

Create `tokens.css` with complete CSS custom property system for dark and light themes per CLAUDE.md specification. Wire into app root so all components can consume semantic tokens.

## Tasks

<task id="1">
Create tokens.css file at apps/web/src/styles/tokens.css with all CSS custom properties from CLAUDE.md design token specification:
- Dark theme (default): --surface-base through --surface-border-subtle, --text-primary through --text-inverse, --shadow-sm/md/lg, --glass-* tokens
- Light theme [data-theme="light"]: All surface, text, shadow, glass tokens
- Shared tokens: --brand-ambient/interactive/hover/muted, --status-critical/warning/success/neutral with backgrounds, --font-family/mono, --space-1 through --space-20, --radius-sm/md/lg/xl/full, --ease-out, --duration-fast/normal/slow, --z-sticky through --z-toast
</task>

<task id="2">
Update apps/web/src/app/globals.css to import tokens.css at the top before Tailwind directives
</task>

<task id="3">
Verify apps/web/src/app/layout.tsx applies data-theme attribute to html or body element for theme switching
</task>

<task id="4">
Run `npm run build` in apps/web to verify no CSS errors
</task>

<task id="5">
Create a simple test: apply --surface-base to a div, verify it renders #111111 in dark mode
</task>
