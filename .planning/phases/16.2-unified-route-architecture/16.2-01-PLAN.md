  <action>
Create `apps/web/src/hooks/usePermissions.ts` implementing the unified permission hook.

Key requirements from spec:
1. Import lens_matrix.json from `@/lib/lens_matrix.json`
2. Accept `lensType: string` parameter (e.g., 'fault', 'work_order')
3. Read user role from `useAuth()` hook
4. Return `PermissionResult` interface with:
   - `can(actionId: string): boolean` - check if user can perform action
   - `allowedActions: string[]` - all actions user can perform
   - `requiresSignature(actionId: string): boolean` - check signature requirement
   - `requiresConfirmation(actionId: string): boolean` - check confirmation requirement

Permission logic:
- If `role_restricted` array is empty: all crew can perform action
- If `role_restricted` array is populated: only listed roles can perform
- All crew roles: `['crew', 'deckhand', 'steward', 'chef', 'eto', 'engineer', 'chief_engineer', 'chief_officer', 'captain', 'manager']`

Also include legacy wrapper functions for backward compatibility:
- `useFaultPermissions()` that calls `usePermissions('fault')` and maps to existing interface
- This allows gradual migration without breaking existing code

Follow the implementation from spec lines 693-816 in UNIFIED-ROUTE-ARCHITECTURE.md.

TypeScript interface:
```typescript
export interface PermissionResult {
  can: (actionId: string) => boolean;
  allowedActions: string[];
  requiresSignature: (actionId: string) => boolean;
  requiresConfirmation: (actionId: string) => boolean;
}
```

  <name>Task 3: Create RouteShell.tsx component</name>
  <files>apps/web/src/components/lens/RouteShell.tsx</files>
  <action>
Create `apps/web/src/components/lens/RouteShell.tsx` implementing the route wrapper component.

Key requirements from spec (lines 199-604 in UNIFIED-ROUTE-ARCHITECTURE.md):

1. **Props interface (RouteShellProps)**:
   - `entityType: EntityType` - maps to lens type
   - `entityId: string` - entity UUID from route params
   - `legacyRedirect?: string` - override redirect when feature flag disabled
   - `listRoute: string` - route for back/close navigation (e.g., '/faults')
   - `pageTitle?: string` - optional title override

2. **EntityType union**:
   ```typescript
   export type EntityType =
     | 'work_order' | 'fault' | 'equipment' | 'part' | 'inventory'
     | 'receiving' | 'certificate' | 'handover' | 'handover_export'
     | 'hours_of_rest' | 'warranty' | 'shopping_list' | 'document' | 'worklist';
   ```

3. **Feature flag guard**:
   - Check `isFragmentedRoutesEnabled()` from `@/lib/featureFlags`
   - If disabled, redirect to `legacyRedirect` or `/app?entity={type}&id={id}`
   - Show "Redirecting..." placeholder during redirect

4. **Data fetching**:
   - Use `useQuery` from `@tanstack/react-query`
   - Fetch from `/v1/entity/{endpoint}/{id}` with Bearer token
   - Map entity type to API endpoint (ENTITY_ENDPOINTS constant)
   - Handle loading, error, and not-found states

5. **Navigation callbacks (route-specific)**:
   - `handleBack`: `router.push(listRoute)`
   - `handleClose`: `router.push(listRoute)`
   - `handleNavigate`: Cross-entity navigation to fragmented route or legacy route
   - `handleRefresh`: Call `refetch()` from useQuery

6. **Rendering**:
   - Wrap in `RouteLayout` from `@/components/layout`
   - Show loading spinner while fetching
   - Show error state with retry button
   - Show 404 state with back-to-list button
   - Delegate to LensContent via internal helper component

7. **LensContent delegation**:
   - Create internal `LensRendererWithRouteCallbacks` component
   - Import all LensContent components (same as LensRenderer.tsx)
   - Pass route-specific callbacks (onBack, onClose, onNavigate, onRefresh)
   - Switch on entityType to render correct component

Reference existing code:
- `LensRenderer.tsx` for component mapping pattern
- `faults/[id]/page.tsx` for loading/error/not-found states (lines 67-106)
- `RouteLayout.tsx` for layout wrapper usage

Constants to define:
- `ENTITY_ENDPOINTS`: Map entity type to API endpoint path
- `ENTITY_LABELS`: Map entity type to human-readable label
- `ENTITY_ROUTES`: Map entity type to fragmented route path
