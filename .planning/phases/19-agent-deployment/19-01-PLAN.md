Create .planning/agents/lens-matrix/ directory. Spawn 12 parallel Task tool agents, each analyzing ONE lens from the codebase.

Each agent receives this prompt template (substitute {LENS_NAME}):
---
You are a Lens Matrix Analyzer for the {LENS_NAME} lens.

**Your job:** Analyze the {LENS_NAME} lens implementation and produce {LENS_NAME}_matrix.json with:

1. **READ filters** - All available filter fields for this lens (status, priority, location, equipment, date ranges, etc.)
2. **MUTATE required_fields** - For each mutation action (create, update, close, etc.), list the required input fields

**Analysis sources:**
- Database schema: supabase/migrations/ (look for pms_{LENS_NAME} table)
- Backend handlers: apps/api/actions/*_handlers.py or supabase/functions/backend_core/actions/handlers/
- Frontend components: apps/web/src/components/*Card.tsx or apps/web/src/app/{LENS_NAME}/

**Output JSON structure:**
```json
{
  "lens": "{LENS_NAME}",
  "read_filters": {
    "status": ["open", "in_progress", "closed"],
    "priority": ["low", "medium", "high", "critical"],
    "location": null,
    "equipment_id": null,
    "date_range": {"field": "created_at", "type": "datetime"}
  },
  "mutate_actions": {
    "create_{LENS_NAME}": {
      "action_id": "create_{LENS_NAME}",
      "required_fields": ["title", "description", "priority"],
      "optional_fields": ["equipment_id", "scheduled_date"],
      "role_restricted": ["captain", "chief_engineer"]
    },
    "update_{LENS_NAME}": {
      "action_id": "update_{LENS_NAME}",
      "required_fields": ["{LENS_NAME}_id", "field_to_update"],
      "optional_fields": ["notes"],
      "role_restricted": []
    }
  }
}
```

**Rules:**
- Analyze ACTUAL codebase files, don't guess
- If a filter doesn't exist, set to null or omit
- For required_fields, only include fields that cause validation errors if missing
- Include role_restricted array per action (empty if no restrictions)
- Write output to .planning/agents/lens-matrix/{LENS_NAME}_matrix.json

Start analysis.
