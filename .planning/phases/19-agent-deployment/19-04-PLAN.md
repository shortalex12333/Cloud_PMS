```typescript
test('READ: show open work orders navigates to /work-orders/status/open', async ({ page }) => {
  // Navigate to app
  await page.goto('http://localhost:3000');

  // Open spotlight search (Cmd+K)
  await page.keyboard.press('Meta+K');

  // Type query
  await page.fill('[data-testid="spotlight-input"]', 'show open work orders');

  // Wait for suggestions
  await page.waitForSelector('[data-testid="suggested-actions"]');

  // Verify Navigate button appears
  const navigateBtn = page.locator('[data-testid="navigate-action"]');
  await expect(navigateBtn).toBeVisible();

  // Click navigate
  await navigateBtn.click();

  // Verify URL
  await expect(page).toHaveURL('/work-orders/status/open');

  // Verify filter chips
  const filterChip = page.locator('[data-testid="filter-chip-status"]');
  await expect(filterChip).toContainText('status: open');
});
```

2. **MUTATE Action Tests (25 tests)**
   - User types MUTATE query → sees Execute suggestion with readiness indicator
   - User clicks Execute → ActionModal opens
   - Modal shows prefilled fields from /prepare
   - User submits → database record created
   - Ledger entry created

Example:
```typescript
test('MUTATE: create high priority work order on ME1 with prefill', async ({ page }) => {
  await page.goto('http://localhost:3000');
  await page.keyboard.press('Meta+K');
  await page.fill('[data-testid="spotlight-input"]', 'create high priority work order on ME1');

  // Wait for action suggestion
  await page.waitForSelector('[data-testid="suggested-actions"]');

  // Verify READY indicator (green checkmark)
  const readyIndicator = page.locator('[data-testid="readiness-indicator"]');
  await expect(readyIndicator).toHaveAttribute('data-state', 'READY');

  // Click execute
  const executeBtn = page.locator('[data-testid="execute-action"]');
  await executeBtn.click();

  // Verify modal opens
  await page.waitForSelector('[data-testid="action-modal"]');

  // Verify prefilled fields
  const equipmentField = page.locator('[data-testid="field-equipment_id"]');
  await expect(equipmentField).toHaveValue('ME1');

  const priorityField = page.locator('[data-testid="field-priority"]');
  await expect(priorityField).toHaveValue('high');

  // Fill remaining required fields
  await page.fill('[data-testid="field-title"]', 'Test work order');

  // Submit
  await page.click('[data-testid="submit-action"]');

  // Verify success toast
  await page.waitForSelector('[data-testid="toast-success"]');

  // Verify database record (using test API)
  const response = await page.request.get('http://localhost:3000/api/test/work-orders?title=Test work order');
  const data = await response.json();
  expect(data.length).toBe(1);
  expect(data[0].equipment_id).toBeTruthy();
  expect(data[0].priority).toBe('high');
});
```

**Common failure patterns and fixes:**

1. **Timing issues:**
   - Replace `page.waitForTimeout(1000)` with `page.waitForSelector('[data-testid="..."]')`
   - Add `{ state: 'visible' }` to waitForSelector calls
   - Wait for network idle before assertions

2. **Selector issues:**
   - If data-testid missing, add to component
   - Use more specific selectors (role + name)
   - Avoid CSS class selectors (theme changes break them)

3. **Database state issues:**
   - Ensure beforeEach creates fresh test yacht
   - Ensure afterEach cleans up test data
   - Use unique identifiers (timestamps) for test records

4. **Auth issues:**
   - Verify test user has correct role for action
   - Check RLS policies allow test operations
   - Use proper auth fixtures

**Flakiness threshold:** <= 5% flake rate (no more than 15 flaky tests out of 300+).

**Fix strategy:**
- Run suite 3 times, identify consistently failing tests
- Fix structural issues (missing selectors, bad cleanup)
- Add retries for network-dependent assertions
- Document any remaining flaky tests with investigation notes
